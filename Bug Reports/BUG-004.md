# 🐞 BUG-004 - Location filter returns properties from incorrect locations

## Overview

| Field | Value |
|-------|-------|
| Project | Real Estate Prediction System |
| Module | Property Search |
| Issue Type | Bug |
| Severity | High |
| Priority | High |
| Status | Open |
| Related Test Case | TC_SEARCH_005 |
| Related Jira Issue | REPT-85 |

---

## Environment

- Browser: Google Chrome 138
- Operating System: macOS Sequoia
- Testing Type: Manual Testing

---

## Preconditions

- User is on the Property Search page.

---

## Steps to Reproduce

1. Open the Search page.
2. Select **Da Nang** as the location.
3. Click **Apply Filter**.

---

## Expected Result

Only properties located in **Da Nang** should be displayed.

---

## Actual Result

Properties from other locations are included in the search results.

---

## Root Cause (Assumption)

The selected location parameter is ignored or incorrectly processed by the search function.

---

## Notes

This defect was identified during manual functional testing as part of the Real Estate Prediction System testing portfolio.
