# 🐞 BUG-005 - System accepts unsupported avatar file format upload

## Overview

| Field | Value |
|-------|-------|
| Project | Real Estate Prediction System |
| Module | User Profile |
| Issue Type | Bug |
| Severity | Medium |
| Priority | Medium |
| Status | Open |
| Related Test Case | TC_PROFILE_006 |
| Related Jira Issue | REPT-86 |

---

## Environment

- Browser: Google Chrome 138
- Operating System: macOS Sequoia
- Testing Type: Manual Testing

---

## Preconditions

- User is logged in and is on the Profile page.

---

## Steps to Reproduce

1. Navigate to the Profile page.
2. Click **Upload Avatar**.
3. Select **avatar.pdf**.
4. Click **Save**.

---

## Expected Result

The system should reject unsupported file formats and display a validation message indicating that only image files are allowed.

---

## Actual Result

The system accepts the PDF file and uploads it successfully.

---

## Root Cause (Assumption)

File type validation is missing or incorrectly implemented during the avatar upload process.

---

## Notes

This defect was identified during manual functional testing as part of the Real Estate Prediction System testing portfolio.
