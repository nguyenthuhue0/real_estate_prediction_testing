# 🐞 BUG-002 - Prediction accepts invalid numeric values without validation

## Overview

| Field | Value |
|-------|-------|
| Project | Real Estate Prediction System |
| Module | Property Prediction |
| Issue Type | Bug |
| Severity | High |
| Priority | High |
| Status | Open |
| Related Test Case | TC_PRE_003 |
| Related Jira Issue | REPT-83 |

---

## Environment

- Browser: Google Chrome 138
- Operating System: macOS Sequoia
- Testing Type: Manual Testing

---

## Preconditions

- User is on the Property Prediction page.

---

## Steps to Reproduce

1. Open the Prediction page.
2. Enter **-50** in the Area field.
3. Fill in all remaining required fields with valid values.
4. Click **Predict**.

---

## Expected Result

The system should validate numeric input and prevent prediction when the value is less than or equal to zero.

---

## Actual Result

The system accepts the invalid value and generates a prediction result.

---

## Root Cause (Assumption)

Input validation for numeric fields is missing before executing the prediction logic.

---

## Notes

This defect was identified during manual functional testing as part of the Real Estate Prediction System testing portfolio.
