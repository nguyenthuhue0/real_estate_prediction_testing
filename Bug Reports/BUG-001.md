# 🐞 BUG-001 - Weak password is accepted during user registration

## Overview

| Field | Value |
|-------|-------|
| Project | Real Estate Prediction System |
| Module | Authentication |
| Issue Type | Bug |
| Severity | Medium |
| Priority | Medium |
| Status | Open |
| Related Test Case | TC_REG_004 |
| Related Jira Issue | REPT-82 |

---

## Environment

- Browser: Google Chrome 138
- Operating System: macOS Sequoia
- Testing Type: Manual Testing

---

## Preconditions

- User is on the Register page.

---

## Steps to Reproduce

1. Navigate to the Register page.
2. Enter a valid Full Name.
3. Enter a valid Email address.
4. Enter a weak password (`123456`).
5. Confirm the password.
6. Click **Register**.

---

## Expected Result

The system should reject weak passwords and display a password strength validation message.

---

## Actual Result

The system accepts the weak password and creates the account successfully.

---

## Root Cause (Assumption)

Password strength validation is missing or not properly implemented during the registration process.

---

## Notes

This defect was identified during manual functional testing as part of the Real Estate Prediction System testing portfolio.
